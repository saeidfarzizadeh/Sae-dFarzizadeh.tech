## Introduction

Self-attention allows models to weigh relationships across all tokens in a sequence simultaneously. Unlike recurrent architectures, transformers trade sequential computation for parallelizable matrix operations — at the cost of quadratic memory in sequence length.

<Callout type="insight">
  Attention is O(n²) in sequence length. This is the central scaling constraint behind sparse attention, linear attention, and state-space models.
</Callout>

## The Core Equation

Given query, key, and value matrices, attention computes a weighted sum of values where weights come from query–key similarity:

```python
import torch

def scaled_dot_product_attention(Q, K, V):
    scores = torch.matmul(Q, K.transpose(-2, -1))
    scores = scores / (Q.size(-1) ** 0.5)
    weights = torch.softmax(scores, dim=-1)
    return torch.matmul(weights, V)
```

## Why It Matters for Invention Work

When designing edge AI systems, understanding attention's compute profile helps you decide whether to distill, prune, or replace attention layers entirely with more efficient alternatives.
